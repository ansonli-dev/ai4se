# AI-Assisted Review Checklist

## Spec Alignment

- [ ] Implementation matches the approved Story and acceptance criteria.
- [ ] No unapproved business rules were invented.
- [ ] No unapproved fields, APIs, events, or permissions were invented.
- [ ] No unrelated scope was added.

## Code Quality

- [ ] Code is readable and maintainable.
- [ ] Responsibilities are clear.
- [ ] Error handling is explicit.
- [ ] Edge cases are handled.
- [ ] Existing patterns are followed.

## Testing

- [ ] Tests verify behavior, not only implementation details.
- [ ] Failure and boundary cases are covered.
- [ ] Contract tests are updated when APIs or events change.
- [ ] Regression tests are added for bug fixes.

## Security And Data

- [ ] Permission checks are correct.
- [ ] Audit logs are correct.
- [ ] Sensitive data is not exposed.
- [ ] No credentials or secrets are included.

## Evidence

- [ ] Verification commands are listed.
- [ ] CI or test evidence is linked.
- [ ] Risks and assumptions are documented.
- [ ] Rollback or recovery notes are present when needed.
