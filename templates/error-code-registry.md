# Error Code Registry

## Naming Rule

Use a stable module prefix and numeric suffix.

Example: `WO-40001` for work-order business validation errors.

## Change Rules

- Do not reuse deleted error codes.
- Do not change semantic meaning after release.
- User-facing messages must be reviewed by product or business owner.
