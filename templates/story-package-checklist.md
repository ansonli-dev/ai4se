# Story Package Checklist

## Before Development

- [ ] Requirement ID and Story ID are assigned.
- [ ] SDD Story Spec is complete.
- [ ] Acceptance criteria use Given/When/Then.
- [ ] AI context boundary is explicit.
- [ ] Technical Spec is created when needed.
- [ ] API contract, event schema, data dictionary, and error code changes are identified.
- [ ] Test Spec is complete.
- [ ] Module Owner is identified.

## During Development

- [ ] AI is used only with approved context.
- [ ] Prompt Card is referenced for internal AI-assisted work.
- [ ] Developer reviews and edits AI output before commit.
- [ ] Tests are written or updated with the implementation.
- [ ] API and data documentation are updated with code changes.

## Before Merge

- [ ] Merge request uses the AI-SDD template.
- [ ] AI usage declaration is complete.
- [ ] Quality gates pass.
- [ ] Test evidence is attached.
- [ ] Module Owner approval is present when required.
- [ ] Known risks and rollback plan are documented.

## After Acceptance

- [ ] Acceptance evidence is linked.
- [ ] Lessons learned are added to the knowledge base.
- [ ] Prompt Card improvements are submitted if internal AI output required significant correction.
- [ ] Metrics are updated.
